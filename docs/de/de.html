Deutsche Lokalisierung für dotProject installieren

  1. Man hole sich die deutschen Lokalisierungsdateien german*.tar.gz von http://sourceforge.net/projects/dotmods/.
  2. Man entpacke den Inhalt des gzipten Archivs in das dotProject-Stammverzeichnis (das Archiv enthält die dotProject-Verzeichnissubstruktur). Folgende Verzeichnisse werden hinzugefügt bzw. bei einer Aktualisierung (teilweise) überschrieben: root_dir/locales/de, root_dir/modules/help/de, root_dir/db/, wobei root_dir das dotProject-Stammverzeichnis repräsentiert. Unter den Windows-Umgebungen ersetze man "/" durch "\".
  3. Will man für alle Benutzer die deutsche Lokalisierung als Standard setzen, melde man sich mit Schreib- und Leserechten für die Systemverwaltung und für die Benutzerverwaltung an (admin). Man wechsle z.B. durch Klick auf System Admin bzw. Systemverwalter in das Systemverwaltungsmodul und wähle dort Default User Preferences bzw. Standardeinstellungen für Benutzer. Dort stelle man Locale bzw. Orts- und Sprachanpassung auf de bzw. Deutsch.
    Alternative zu dem gerade beschriebenen Vorgehen: Man ändere in root_dir/includes/config.php die Konfigurationsvariable zu $dPconfig['host_locale'] = "de";, um Deutsch als Standardsprache einzustellen. Dann wird bereits die Anmeldungsseite auf deutsch angezeigt.
  4. Will man als Administrator für einen einzelnen Benutzer die deutsche Lokalisierung als Standard setzen, melde man sich mit Schreib- und Leserechten für die Systemverwaltung und für die Benutzerverwaltung an (admin). Man wechsle z.B. durch Klick auf User Admin bzw. Benutzerverwalter in das Benutzerverwaltungsmodul und wähle dort den gewünschten Benutzer aus der Tabelle. Auf der Detailsseite für diesen Benutzer wähle man Edit User Preferences bzw. Einstellungen bearbeiten aus den Brotkrümeln. Dort stelle man Locale bzw. Orts- und Sprachanpassung auf de bzw. Deutsch.
  5. Einige Listeneinträge werden in dotPoject über die zugrunde liegende Datenbank organisiert. Man nennt diese Einträge Systemweite Werte oder System Values. Will man diese ebenfalls in der deutschen Sprache angezeigt bekommen, so wende man das mitgelieferte Datenbank-Patch mit dem Namen root_dir/db/sysvals_de.sql an. Dies kann man im Allgemeinen auf zwei Arten tun: Entweder relativ bequem über phpMyAdmin, einer Weboberfläche zur Verwaltung der MySQL-Datenbanken, oder per Befehlseingabe via Kommandozeile direkt an den Datenbankserver - dies geht ohne zusätzliche Software immer. Vor Veränderungen der Datenbank, empfiehlt es sich immer, eine Sicherungsdatei der Datenbank anzulegen. Dieser Vorgang ist unter Weg B beschrieben, ist aber auch über den ersten Weg möglich - dazu entnehme man die Vorgehensweise bitte der Dokumentation zu phpMyAdmin.
    A Man starte phpMyAdmin in einem Webbrowser und wähle die entsprechende Datenbank (zumeist dotproject oder etwas ähnliches). Dann wähle man 'SQL' und lade dort über das 'Textfile'-Feld die auszuführende Datei. Die Datenbank sollte nun aktualisiert sein. Die beschriebene Vorgehensweise bezieht sich auf phpMyAdmin Version 2.5.1, andere Versionen können von dieser Vorgehensweise abweichen; in diesem Fall entnehme man bitte relevante Informationen bitte der phpMyAdmin-Dokumentation. PhpMyAdmin ist wie dotProject OpenSource-Software, d.h. es ist frei erhältlich. Es ist ebenfalls ein SourceForge-Projekt, man erhält es unter .
    B Man benutzt die Kommandozeile, um direkt Befehle an den Datenbankserver zu geben. Die nachfolgenden Beispiele gehen von dem Standarddatenbanknamen dotproject aus. Gegebenenfalls müssen die Befehle entsprechend angepasst werden, d.h. man ersetze 'dotproject' durch den dotProject-Datenbanknamen der vorliegenden Installation. Um herauszufinden, wie der Datenbankname lautet, schaue man sich - eine funktionierende dotProject Installation vorausgesetzt - den Eintrag der Konfigurationsvariable $dPconfig['dbname'] = "gesuchterName"; in der dotProject-Konfigurationsdatei 'root_dir/includes/config.php' an. Hier wäre gesuchterName der gewünschte dotProject-Datenbankname.
    Vor solchen Veränderungen an der (Live-)Datenbank, empfiehlt es sich immer, zuerst eine Sicherungsdatei mit dem Datenbankinhalt zu erstellen. Für die beiden nachfolgenden Befehlseingaben wechsle man in der Kommandozeile in das Verzeichnis root_dir/db/, sofern man sich nicht schon dort befindet. Ausserdem gebe man nach -u den entsprechenden MySQL-Benutzernamen und nach -p das korrespondierende MySQL-Passwort für die dotProject-Datenbank an.
     
    Erstellung einer Sicherungskopie einer bestehenden Datenbank
    mysqldump -udp_user -pdp_pass dotproject > dp100_backup.sql

    Aktualisierung der bestehenden Datenbank
    mysql -udp_user -pdp_pass dotproject < sysvals_de.sql

In dem Abschnitt Aktualisierung der Dokumentation findet man weitere Erklärungen und Hilfestellungen zu oben angeführten Verfahren, z.B. insbesondere zur Datenbankverwaltung.
 
Das deutsche Übersetzungsteam freut sich über Anregungen, Kritik und weitere Teilnahme zur Verbesserung der Übersetzung. Man besuche dazu http://sourceforge.net/projects/dotmods/.
